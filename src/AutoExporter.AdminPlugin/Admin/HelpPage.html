Auto Exporter

Automate XProtect (TM) video exports on a recurring rule or on demand, and keep them on disk within a bounded size and age. Exports are produced by a standalone agent service running on the machine that stores them.

How the pieces fit.
  • An agent service runs on each export machine. It signs in to Milestone and does the actual exporting.
  • A tray app on that machine sets the Milestone connection, the export folder, and the size and age limits.
  • This plugin (here in the Management Client) defines the jobs, assigns each job to an agent, and shows status. It does not export anything itself.

The tree

  • Overview is one page with three sections, Agents, Jobs, and Executions. The sections refresh themselves, so there is no Refresh button.
  • Jobs is also a node so the Rules engine can target a specific job. The Jobs node and the Overview Jobs section edit the same jobs.

Agents

An agent appears here automatically once its service signs in (it registers itself by hostname). You do not add agents by hand. The Agents section shows each agent's status, version, export folder, and its size and age limits (set in that machine's tray app).

  • An agent shows Offline once it has missed several heartbeats or its service was stopped or uninstalled.
  • An offline agent can be removed with Remove agent. This also deletes every job assigned to that agent, so no job is left pointing at a missing agent. A live agent cannot be removed because it would just register itself again.

Set up a job

  1. Open Jobs (or the Overview Jobs section) and add a job.
  2. Give it a name, pick the agent that runs it, choose the format (XProtect or AVI), set the time range, and add the cameras or camera groups.
  3. Save. A job cannot be saved without a name, an agent, and at least one camera or group.
  4. Use Run now in the Jobs section to test it, or build an XProtect Rule with the Run Auto Export Job action. Watch the Executions section for the result.

Where exports go and how long they stay

The export folder, the maximum size, and the retention in days are per agent, not per job. They are set in the agent's tray app, not here. Each job writes into its own subfolder under that export folder, and each run gets its own timestamped folder.

<Export folder>\
  <Job name>\
    28.05.2026_0300\
      <Camera name>\
        ...exported files...
    28.05.2026_0400\
      ...

Old runs are removed oldest first to stay under the size limit, and runs older than the retention are removed as well. A limit of 0 means unlimited.

Time range

Each job exports Last N [Minutes, Hours, Days, or Months]. The range is anchored at the moment the job runs, so the same job works for a scheduled rule, a manual Run now, or an event-driven rule.

Run timeRangeExported window
2026-05-28 03:00Last 2 Days2026-05-26 03:00 to 2026-05-28 03:00
2026-05-28 14:22Last 2 Hours2026-05-28 12:22 to 2026-05-28 14:22

Months counts as 30 days.

Disabled cameras

The camera picker hides disabled cameras, and the agent skips a disabled camera if it is still on a job (for example a group member that was later disabled). A disabled camera never fails the whole job. If every selected camera is disabled or has no data in the range, the run is recorded as Skipped, not Failed, with the camera names in the detail.

Recording server reachability

When an agent signs in, it checks that it can reach every recording server. If one cannot be reached (often bad DNS, a missing hosts entry, or a firewall), the agent's tray app warns about it. Exports for cameras on an unreachable recorder will fail until it can be reached.

Rule action and events

The plugin exposes one action, Run Auto Export Job, which you point at a specific job. It raises three events you can react to in rules:

  • Auto Export: Job Started fires when a run begins.
  • Auto Export: Job Succeeded fires when a run finishes.
  • Auto Export: Job Failed fires on a hard error.
Password storage. An export encryption password is stored in the Milestone configuration. Use encryption only where that is acceptable, and restrict configuration database access with Milestone roles. AVI does not support encryption.

Status and logs

The Executions section shows recent runs from every agent (newest first) with the outcome, camera count, size, and detail. Each agent keeps its own run history and log files under %ProgramData%\MSCPlugins\AutoExporter\ on its machine. The tray app can open those logs.

Troubleshooting

  • No agent to pick when adding a job. Start the agent service on the export machine and connect it in the tray app. It will appear in Agents within a few seconds.
  • A camera has no recordings in the range. Cameras with no data in the window are skipped. The run is recorded as Partial when some cameras still exported, or Skipped when none did, with the names in the detail. This is informational, not an error.
  • AVI plus encryption was rejected when saving. AVI does not support encryption. Switch to XProtect or turn encryption off.
  • A recording server is reported unreachable. Check name resolution and the firewall from the agent machine to that recorder.
